Heuristics:

The pulse sensor used is not the most accurate one in the world but does give values that are good enough for basic health monitoring and modeling. The step counter algorithm used is also not very accurate as you may need to multiply the values by 2 to get a more accurate result and also the change of acceleration required to be considered a step might hinder the accuracy but the algorithm is adequate for a step counter and does not have to be perfect.

Evaluating how my project met the user requirements:

Functionality: In my project, I ensured that the system performed specific functions essential for health monitoring and fitness tracking, such as data collection from the pulse sensor (BPM) and Micro:bit (steps) visualization of BPM trends using Matplotlib graphs, and communication with the user through prompts and feedback mechanisms (questionnaire).

Usability: I made sure users can easily input their health-related data, and receive feedback without encountering any significant usability challenges or having to answer silly questions.

Performance: I designed the system to operate smoothly and efficiently, ensuring speed, responsiveness, and reliability in data collection, analysis, and visualization processes. This helped meet user expectations regarding the system's performance, providing a seamless experience. The delay placed on every operation is just about right and is not too fast.

Security: In terms of security, Google Sheets uses end-to-end encryption for data transmission and storage, utilizing HTTPS for encrypted communication and encrypting data at rest on Google's servers. With compliance certifications such as ISO 27001 and GDPR your data is definitely safe. This means that the user doesn´t need to worry about data handling as the user is kept anonymous and no personal information is collected.

Customized response: I also incorporated some degree of customization. For example, users could input their personal health data, such as weight and height, to receive personalized feedback and recommendations tailored to their individual needs and preferences.

What I could have done differently:

I could have had a better and more accurate algorithm for step counting by having it use the accelerometer in a more sophisticated way where the percentage error would be less than 2%. I could have used a better and more expensive pulse sensor that would greatly affect the accuracy of the BPM. I would have also loved to change the UI user to a more colorful and nicer looking UI. I would have also liked to be able to still have the user get prompted to have the user tell the program if they met their daily goal even if they said no when I asked them if they wanted a graph to be generated.
